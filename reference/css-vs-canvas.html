The one question

Is position a function of time, or an integration of state?

pos = f(t) → CSS can do it.  ·  pos += vel·dt with vel changing each frame → needs a JS loop.

The canvas loop (the pattern to memorise)

function loop() { requestAnimationFrame(loop); // schedule next frame const dt = (now - last) / 1000; // seconds since last frame update(dt); // advance STATE: pos += vel·dt, etc. draw(); // repaint from current state }
  • dt-based, not per-frame constants: multiply motion by dt so speed is the same on 60Hz and 144Hz screens.
  • update / draw split: one mutates state, one only reads it to paint. Keeps physics independent of rendering.
  • Damping: v *= exp(-rate·dt) — framerate-independent friction; the “memory” that makes spring-back work.

Reach-for ladder

  • Function of time → CSS @keyframes / transition.
  • State, few elements → JS loop on DOM/SVG transforms (or Motion).
  • State, many elements → 2D <canvas> (this is grid.html).
  • State, huge counts / per-pixel → WebGL / shaders.
